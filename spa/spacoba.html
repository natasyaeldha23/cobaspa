Progressive Web Apps

Pada bulan-bulan ini Google mengadakan kampanye penggunaan PWA. Apakah sebenarnya PWA itu ? PWA sendiri merupakan kependekan dari Progressive Web Apps. Dari katanya kita melihat adanya unsur web dan apps, mungkin kita bisa menebak bahwa Google berusaha menjembatani antara web application dan mobile application yang biasanya berjalan pada Android dan Ios. Para pengembang aplikasi saat ini terbagi menjadi dua yaitu web based atau app based. Pada web based mengandalkan web server di mana manajemen lebih mudah karena semua proses utama berada di server. Client mengandalkan browser dan harus terhubung dengan jaringan supaya dapat menjalankan web application. Pada app based terlihat lebih cepat karena tidak selalu terhubung dengan jaringan, tetapi development maupun pemeliharaan aplikasi bisa berlangsung cukup rumit. Kita mungkin tahu bahwa saat melakukan pengembangan mobile application kita harus membuat pembuatan terpisah antara untuk Android dan juga Ios. Pada saat pengembangan dengan target Androidpun kita juga harus memperhatikan versi dari Android di mana aplikasi kita akan dideploy. Selain itu kita harus membuat mekanisme update. Di sini Google membuat PWA yang menjembatani antara kemudahan manajemen web application dan kehandalan mobile application. Dengan menggunakan browser Chrome dan Javascript, kita akan menggunakan cache dalam melakukan interaksi dengan pengguna aplikasi saat kondisi jaringan sedang memburuk. PWA atau Progressive Web Apps menawarkan tiga hal utama yaitu :

1. Reliable Aplikasi akan cepat dijalankan bahkan pada kondisi jaringan internet yang lambat.

2. Fast Berespon dengan cepat dalam berinteraksi dengan pengguna aplikasinya.

3. Engaging Menyerupai mobile application, pengguna akan nyaman menggunakan aplikasi berbasis PWA

Where you can learn and measure

Whether you already have a website or you're just getting started, learn to build for the modern web at web.dev. Then, apply your new skills and solutions to any personal or professional site you work on. After you've launched your site, come back to web.dev and measure how well it supports your users. If there are areas where it can improve, you'll get immediate steps to increase your metrics. As the web advances, users' expectations get higher. With web.dev's guidance, you can give your users the best possible experience, wherever they are.
